◆ Default()

static const StartupMode paimon::StartupMode::Default ( )
static

Determines actual startup mode according to other table properties.

If "scan.snapshot-id" is set the actual startup mode will be "from-snapshot", otherwise the actual startup mode will be "latest-full".

◆ LatestFull()

static const StartupMode paimon::StartupMode::LatestFull ( )
static

For streaming sources, produces the latest snapshot on the table upon first startup, and continue to read the latest changes.

For batch sources, just produce the latest snapshot but does not read new changes.

◆ Latest()

static const StartupMode paimon::StartupMode::Latest ( )
static

For streaming sources, continuously reads latest changes without producing a snapshot at the beginning.

For batch sources, behaves the same as the "latest-full" startup mode.

◆ FromSnapshot()

static const StartupMode paimon::StartupMode::FromSnapshot ( )
static

For streaming sources, continuously reads changes starting from snapshot specified by "scan.snapshot-id", without producing a snapshot at the beginning.

For batch sources, produces a snapshot specified by "scan.snapshot-id" but does not read new changes.

◆ FromSnapshotFull()

For streaming sources, produces from snapshot specified by "scan.snapshot-id" on the table upon first startup, and continuously reads changes.
